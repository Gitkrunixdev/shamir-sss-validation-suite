# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in this repository, please report it privately.

- Use GitHub's private vulnerability reporting feature (enabled).
- Alternatively, contact: shamircrackerlab@protonmail.com
- Do not open public issues for security concerns.

We aim to respond within 48 hours and resolve valid reports promptly.

## Scope

This repository is intended for validation and educational purposes only.  
It does not handle production secrets or live data.

## Disclaimer

This codebase does not log sensitive data in plaintext.  
All secrets are masked or synthetic.  
Use responsibly and review before deploying in production.
